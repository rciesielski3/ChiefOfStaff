# QA-News Sync & Aggregation Architecture

## Overview

QA-News provides three complementary views of the curated article knowledge base generated by the Daily Brief pipeline:

1. **Daily View** — Top 50 most recent articles (fresh data)
2. **Weekly View** — All articles grouped by ISO week with auto-generated summaries
3. **Monthly View** — Top 25 curated articles per month with auto-generated summaries

All data is auto-generated from the canonical article store via scheduled exports and synced to the Next.js application's local data directory for real-time consumption. Summaries are generated using Claude Opus, capturing themes and key insights from article titles and categories.

### Core Design Principles

- **Automated Generation** — No manual curation required; exports run after daily brief completes
- **Local Data Sync** — Exports written to JSON files, synced to qa-news/data/ for Next.js consumption
- **Smart Summaries** — Claude Opus scans article titles and domain categories to generate natural language summaries (1-2 sentences)
- **Fallback Resilience** — If API is unavailable, summaries default to "{weekOf/monthOf}: N articles"
- **Low Cost** — Summaries cost ~$0.10-0.20/day; exports run on simple grouping logic
- **Version Control** — All exports committed to GitHub for audit trail

## Data Flow

```
Daily Brief Workflow (runs at 08:00 UTC)
  ↓
  Produces: canonical_articles.ndjson
  ↓
Export QA-News Workflow (runs at 08:05 UTC)
  ├─ export-latest-news.ts
  │   └─ Extracts top 50, writes to latest.json
  ├─ export-weekly-highlights.ts
  │   ├─ Groups articles by ISO week
  │   ├─ Generates weekly summary via Claude Opus
  │   └─ Writes to weekly-highlights.json
  └─ export-monthly-recap.ts
      ├─ Groups articles by month
      ├─ Curates to top 25 per month
      ├─ Generates monthly summary via Claude Opus
      └─ Writes to monthly-recap.json
  ↓
Sync to qa-news/data/ (via GitHub Actions)
  ↓
Next.js App Loads Local Files
  ├─ GET /api/latest → qa-news/data/latest.json
  ├─ GET /api/weekly → qa-news/data/weekly-highlights.json
  └─ GET /api/monthly → qa-news/data/monthly-recap.json
  ↓
User Views Daily / Weekly / Monthly Pages
```

## Weekly Highlights

### Algorithm

Groups all articles by ISO week (Monday as week start). Each week includes every article published during that 7-day period, sorted by publish date (newest first).

### Data Schema

```json
{
  "weekOf": "2026-07-13",
  "summary": "Key themes across AI breakthroughs, enterprise adoption, and security concerns.",
  "articleCount": 42,
  "items": [
    {
      "id": "article-1234",
      "title": "OpenAI Announces GPT-5 Prototype",
      "url": "https://...",
      "summary": "Brief summary...",
      "domain": "AI Governance",
      "publishedAt": "2026-07-18T14:22:00Z"
    },
    ...
  ]
}
```

### Summary Generation

1. Extracts article titles and domain categories for the week
2. Sends to Claude Opus with prompt: "Summarize the week's news themes in 1-2 sentences based on these titles and categories"
3. Stores result in `summary` field
4. If API unavailable, falls back to: `"Week of {weekOf}: {articleCount} articles"`

### Example Weekly Entry

```json
{
  "weekOf": "2026-07-20",
  "summary": "Week focused on AI safety frameworks and regulatory developments, with major updates in open-source model releases.",
  "articleCount": 38,
  "items": [...]
}
```

## Monthly Recap

### Algorithm

Groups all articles by month, curates each month to top 25 articles (sorted by publication relevance, then date). Months are indexed by first day of month (YYYY-MM-01).

### Data Schema

```json
{
  "monthOf": "2026-07-01",
  "summary": "July marked a turning point in AI governance with EU AI Act enforcement and major open-source model releases.",
  "articleCount": 25,
  "totalForMonth": 187,
  "items": [
    {
      "id": "article-5678",
      "title": "EU AI Act: First Compliance Checks Begin",
      "url": "https://...",
      "summary": "Brief summary...",
      "domain": "Regulation & Policy",
      "publishedAt": "2026-07-15T09:11:00Z"
    },
    ...
  ]
}
```

### Summary Generation

1. Extracts titles and categories from all articles in the month
2. Sends to Claude Opus with prompt: "Summarize July's major news themes in 1-2 sentences based on these titles and categories"
3. Stores result in `summary` field
4. Includes `totalForMonth` (total articles for reference) and `articleCount` (curated to 25)
5. If API unavailable, falls back to: `"Month of {monthOf}: {articleCount} articles ({totalForMonth} total)"`

### Example Monthly Entry

```json
{
  "monthOf": "2026-07-01",
  "summary": "July saw significant AI governance developments, with the EU AI Act moving into enforcement phase and major advances in open-source model availability.",
  "articleCount": 25,
  "totalForMonth": 187,
  "items": [...]
}
```

## Running Exports Manually

### Quick Export All

```bash
npm run export-qa-news
```

This runs all three export CLI commands and syncs to qa-news/data/.

### Export Individual Views

```bash
# Export latest 50 articles
npx ts-node src/cli/export-latest-news.ts > latest.json

# Export weekly highlights with summaries
npx ts-node src/cli/export-weekly-highlights.ts > weekly-highlights.json

# Export monthly recap with summaries
npx ts-node src/cli/export-monthly-recap.ts > monthly-recap.json

# Copy all exports to qa-news data directory
cp latest.json weekly-highlights.json monthly-recap.json ../qa-news/data/
```

### With Environment Variables

If running manually (not via GitHub Actions):

```bash
export ANTHROPIC_API_KEY="sk-ant-..."

npm run export-qa-news
```

Summaries require the API key. Without it, summaries default to fallback format.

## Summaries

### How Summaries Work

1. **Extraction** — Collects article titles and domain categories for the period (week/month)
2. **Prompt** — Sends to Claude Opus with task: "Summarize the following themes in 1-2 natural language sentences"
3. **Generation** — Opus returns theme-focused summary (e.g., "Week saw major AI safety updates and enterprise adoption announcements")
4. **Fallback** — If API unavailable or rate-limited, summary becomes `"{weekOf/monthOf}: N articles"`

### Configuration

- **Model** — Claude Opus (high quality for synthesis)
- **Temperature** — 0.7 (balanced creativity and consistency)
- **Cost** — ~$0.10-0.20/day for all three exports
- **Environment** — Requires `ANTHROPIC_API_KEY` (from .env)

### Quality Considerations

- Summaries generated from metadata (titles + categories), not full article text, to keep costs low
- Summaries are deterministic given the same input (same titles → same summary)
- If articles change, summaries regenerate on next export run
- Summaries are human-readable and suitable for UI display

### Disabling Summaries

To run exports without summaries (faster, zero cost):

```bash
# Set flag in export command or use fallback-only mode
# (Currently not configurable; would require code change)
```

To disable in the future, modify export services to skip Opus call and use fallback format directly.

## Troubleshooting

### Problem: Stale Data in qa-news/data/

**Symptoms:** Weekly or monthly pages show old articles, not recent ones.

**Causes:**
- Export workflow didn't run (check GitHub Actions logs)
- Exports ran but sync step failed
- Manual exports not copied to qa-news/data/

**Solutions:**
1. Check GitHub Actions: `.github/workflows/export-qa-news.yml`
   - Verify workflow is enabled
   - Check last run status
   - Review logs for errors
2. If workflow OK, manually re-run:
   ```bash
   npm run export-qa-news
   ```
3. Verify files exist in qa-news/data/:
   ```bash
   ls -la ../qa-news/data/
   ```
4. Commit and push updated files:
   ```bash
   git add ../qa-news/data/*.json
   git commit -m "chore: refresh qa-news data"
   git push
   ```

### Problem: Empty or "N articles" Summaries

**Symptoms:** Weekly/monthly summaries show only "{weekOf}: 42 articles" instead of theme-focused text.

**Causes:**
- `ANTHROPIC_API_KEY` not set in GitHub Actions secrets
- API rate limit exceeded (temporary)
- Network timeout during summary generation

**Solutions:**
1. Check GitHub Actions secrets:
   - Verify `ANTHROPIC_API_KEY` is set in repo secrets
   - Verify key is valid (test locally first)
2. Test locally:
   ```bash
   export ANTHROPIC_API_KEY="sk-ant-..."
   npx ts-node src/cli/export-weekly-highlights.ts
   ```
   - Should show themed summaries, not fallback format
3. Check API status (anthropic.com)
4. If persistent, temporarily disable summaries:
   - Modify export service to skip Opus call
   - Summaries will use fallback format (still useful for users)
   - Re-enable when API is stable

### Problem: Slow Exports or Timeouts

**Symptoms:** Export CLI commands take >30 seconds or timeout in GitHub Actions.

**Causes:**
- Large canonical_articles.ndjson (thousands of articles)
- Slow network (fetching summaries from Opus)
- GitHub Actions runner resource constraints

**Solutions:**
1. Check file size:
   ```bash
   wc -l data/canonical_articles.ndjson
   ```
   - If >5000 articles, performance is expected to degrade
2. Profile locally:
   ```bash
   time npx ts-node src/cli/export-latest-news.ts > /dev/null
   ```
   - Export-latest should complete in <2 seconds
   - Export-weekly should complete in <5 seconds
   - Export-monthly should complete in <5 seconds
3. If slow, check for:
   - Blocking I/O (file reads)
   - Unbounded loops (grouping logic)
   - Missing indices or caches
4. Optimize:
   - Cache grouped results in memory
   - Parallelize summary generation (batch requests)
   - Increase GitHub Actions timeout (workflow .yml)
5. If still slow, disable summaries temporarily:
   - Use fallback format for summaries
   - Bring back summaries in off-hours

## Architecture Decisions

### Why Local Sync vs. API?

**Decision:** Sync exports to qa-news/data/ as JSON files for Next.js to load locally.

**Rationale:**
- Simpler deployment (no separate API server needed)
- Faster page loads (local file I/O vs. HTTP)
- Works offline or with unreliable network
- Data versioned in Git (audit trail)
- Reduces operational complexity

**Alternative Considered:** Real-time API endpoint in ChiefofStaff backend.
- Rejected: Adds server-side code and deployment complexity
- Sync approach is sufficient for daily/weekly/monthly cadence

### Why Three Views?

**Decision:** Daily (top 50), Weekly (all), Monthly (top 25).

**Rationale:**
- Daily: Fresh, trending articles (curated for quality)
- Weekly: Comprehensive view of week's coverage (all articles)
- Monthly: Curated themes and highlights (top stories only)
- Three views serve different user needs (latest news, comprehensive, highlights)

### Why Claude Opus for Summaries?

**Decision:** Use Claude Opus (high-quality model) for theme extraction.

**Rationale:**
- Better quality summaries than smaller models
- Cost still reasonable ($0.10-0.20/day)
- One summary per week/month (not per article), so cost scales slowly
- Fallback to static text if API unavailable (no hard dependency)
