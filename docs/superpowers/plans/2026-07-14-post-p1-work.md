- [ ] **Step 1: Create CI workflow for testing on PRs**

Create `.github/workflows/test.yml`:

```yaml
name: Test

on:
  pull_request:
    paths:
      - 'qa-news/**'
      - '.github/workflows/test.yml'

jobs:
  test:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: qa-news
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: 'qa-news/package-lock.json'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm test -- --coverage
      
      - name: Upload coverage
        uses: codecov/codecov-action@v3
        with:
          files: ./qa-news/coverage/lcov.info
          flags: unittests
          fail_ci_if_error: false
```

This workflow:
- Triggers on PR when qa-news/ or workflow itself changes
- Runs on latest Ubuntu
- Sets up Node 20 with caching
- Installs dependencies
- Runs full test suite
- Uploads coverage (optional, for visibility)

- [ ] **Step 2: Create deploy workflow for GitHub Pages**

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy QA News to GitHub Pages

on:
  push:
    branches:
      - main
    paths:
      - 'qa-news/**'
      - '.github/workflows/deploy.yml'

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: qa-news
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: 'qa-news/package-lock.json'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm test
      
      - name: Build static site
        run: npm run build
      
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v2
        with:
          path: 'qa-news/out'
  
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v2
```

This workflow:
- Triggers on push to main when qa-news/ or workflow changes
- Runs tests (quality gate before deploy)
- Builds static Next.js export
- Uploads artifact to GitHub Pages
- Deploys to configured custom domain (qa-news.rcieskelski.dev)
