# Task 3 Brief: Add validation checkpoint that reports but doesn't block

**Part of:** Workflow Resilience plan (docs/superpowers/plans/2026-07-23-fix-workflow-failures.md)

**Goal:** Add non-blocking validation checkpoints to the daily-brief workflow so it reports issues but never fails on recoverable errors, allowing the pipeline to reach completion even when data quality issues exist.

**Problem:** The daily-brief workflow currently stops when certain steps fail. Even after Task 1 and Task 2 fixes, the workflow could still stop if evolution or insights fail. Need resilient checkpoints that log warnings but exit 0.

**What to Build:**
1. Update `.github/workflows/daily-brief.yml` to add non-blocking validation steps
2. Add `continue-on-error: true` or use `if: always()` on critical steps
3. Create validation checkpoint step that checks fact files exist and have data
4. Report summary showing which stages succeeded/warned
5. Ensure workflow completes to end (no early exits) on recoverable errors

**Files to Touch:**
- Modify: `.github/workflows/daily-brief.yml` (add validation checkpoints + error handling)

**Global Constraints:**
- Workflow syntax valid (no YAML errors)
- All critical steps use `if: always()` or `continue-on-error: true`
- Non-critical failures exit 0 (graceful)
- Genuine errors (missing API key, out of disk) still fail the workflow
- No changes to step logic, only error handling

**Implementation Details:**

### Current Workflow Flow

The workflow has these steps:
1. Generate Daily Brief (fetch RSS)
2. Verify Extraction Prerequisites
3. Extract Knowledge Facts
4. Verify Extraction Output
5. Run knowledge evolution
6. Generate insights
7. Check insights generated
8. Report insights
9. Commit changes
10. Create Pull Request
11. Enable auto-merge

### Changes Needed

1. **Add continue-on-error to evolve-knowledge step** (around line 92):
   ```yaml
   - name: Run knowledge evolution
     if: always()
     id: evolve
     run: |
       echo "Starting knowledge evolution..."
       npx ts-node src/scripts/evolve-knowledge.ts || echo "⚠️  Knowledge evolution encountered warnings but continuing"
       echo "✅ Knowledge evolution step completed"
     continue-on-error: true
   ```

2. **Add validation checkpoint after evolution** (new step):
   ```yaml
   - name: Validate evolved facts
     if: always()
     run: |
       if [ ! -f "data/knowledge_facts.ndjson" ]; then
         echo "⚠️  Facts file not found after evolution, but continuing"
         exit 0
       fi
       
       FACT_COUNT=$(wc -l < data/knowledge_facts.ndjson || echo 0)
       
       if [ "$FACT_COUNT" -eq 0 ]; then
         echo "⚠️  No facts available after evolution, but continuing"
         exit 0
       fi
       
       echo "✅ Facts validation: $FACT_COUNT facts ready for processing"
   ```

3. **Verify all critical steps have error handling:**
   - Extract Knowledge: check has `if: always()` ✅
   - Generate Insights: check has `if: always()` ✅
   - Report Insights: check has `continue-on-error: true` ✅
   - (Update any that don't)

4. **Add final summary step** (optional but recommended):
   ```yaml
   - name: Workflow summary
     if: always()
     run: |
       echo "Daily Brief Workflow Summary:"
       echo "- Extraction: ${{ steps.verify-extraction.outcome }}"
       echo "- Evolution: ${{ steps.evolve.outcome }}"
       echo "- Insights: ${{ steps.generate-insights.outcome }}"
       echo "✅ Workflow completed (failures non-blocking)"
   ```

### Key Principles

- `if: always()`: Step runs even if previous steps failed
- `continue-on-error: true`: Step failing doesn't stop workflow
- `exit 0` in validation: Always succeed on data checks
- Exit 0 on warnings, exit 1 only on genuine crashes
