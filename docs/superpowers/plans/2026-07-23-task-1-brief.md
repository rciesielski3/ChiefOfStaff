# Task 1 Brief: Make evolve-knowledge resilient to JSON parsing errors

**Part of:** Workflow Resilience plan (docs/superpowers/plans/2026-07-23-fix-workflow-failures.md)

**Goal:** Fix evolve-knowledge to gracefully skip malformed JSON facts instead of crashing, allowing the pipeline to continue.

**Problem:** Currently when a single fact line contains invalid JSON, `JSON.parse()` throws and the entire workflow fails. Need to make this non-blocking.

**What to Build:**
1. Modify `src/scripts/evolve-knowledge.ts` to parse facts line-by-line, skipping malformed JSON with warnings
2. Add test file `tests/scripts/evolve-knowledge.test.ts` to verify recovery behavior
3. Report how many facts were skipped due to parse errors

**Files to Touch:**
- Modify: `src/scripts/evolve-knowledge.ts` (lines 62-76 for parsing, lines 104-130 for logging)
- Create: `tests/scripts/evolve-knowledge.test.ts`

**Global Constraints:**
- TypeScript strict mode (no `any`)
- All tests passing before commit
- Commit prefix: `fix:` (no AI co-author)
- TDD approach: failing test → implementation → green test

**Implementation Details:**

See plan for full test code (tests should use temp fixtures via environment variables).

### Step 2: Modify evolve-knowledge.ts
Replace lines 62-76 that currently do:
```typescript
const inputFacts: KnowledgeFact[] = factLines.map((line, index) => {
  try {
    return JSON.parse(line) as KnowledgeFact;
  } catch (error) {
    console.warn(...);
    throw error;  // <-- THIS THROWS, BLOCKING
  }
});
```

With graceful skipping:
```typescript
const inputFacts: KnowledgeFact[] = [];
let skippedCount = 0;

for (let index = 0; index < factLines.length; index++) {
  try {
    const fact = JSON.parse(factLines[index]) as KnowledgeFact;
    inputFacts.push(fact);
  } catch (error) {
    skippedCount++;
    console.warn(`[Knowledge Evolution] Warning: Failed to parse fact at line ${index + 1}, skipping`);
  }
}

if (skippedCount > 0) {
  console.warn(`[Knowledge Evolution] ⚠️  Skipped ${skippedCount} facts due to parse errors`);
}
```
