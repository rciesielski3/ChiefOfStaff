# M5 Planning: Weekly Highlights and Monthly Recap

> **For agentic workers:** This is the specification for M5. The companion implementation plan is `docs/superpowers/plans/2026-07-14-m5-implementation.md`. Execute with superpowers:subagent-driven-development using parallel tracks A and B.

## Goal

Extend QA News with aggregated views (Weekly Highlights, Monthly Recap) providing curated summaries of articles over time periods.

M3 provides daily articles; M4 aggregates them into the Knowledge Layer (`canonical_articles` Data Table). M5 aggregates that same canonical data by week and by month, surfacing the highest-signal articles for readers who want a periodic digest rather than the daily firehose.

## Architecture

- Weekly/Monthly aggregation workflows (n8n)
- JSON exports (`weekly-data.json`, `monthly-data.json`) to `qa-news/public/`
- QA News pages for `/weekly` and `/monthly` routes
- Data source: `canonical_articles` Data Table (M4 Knowledge Layer)

Data flow:

```
canonical_articles (M4 Data Table)
        │
        ├─► aggregate-weekly (n8n)  ─► qa-news/public/weekly-data.json  ─► /weekly page
        │
        └─► aggregate-monthly (n8n) ─► qa-news/public/monthly-data.json ─► /monthly page
```

Both aggregation workflows read the same source table, so they are fully independent and can run and be developed in parallel.

## Data Model

### Weekly Highlights

```json
{
  "week": "2026-W28",
  "weekStart": "2026-07-07",
  "weekEnd": "2026-07-13",
  "articles": [],
  "summary": "Optional week overview",
  "generatedAt": "2026-07-14T12:00:00Z"
}
```

- `week` — ISO 8601 week identifier (`YYYY-Www`)
- `weekStart` / `weekEnd` — inclusive date bounds of the aggregation window (ISO date)
- `articles` — top 10-15 canonical articles ranked by score (DESC), deduplicated by article id
- `summary` — optional human/LLM-generated week overview (may be empty string)
- `generatedAt` — ISO 8601 timestamp of export (cache-busting signal)

### Monthly Recap

```json
{
  "month": "2026-07",
  "monthStart": "2026-07-01",
  "monthEnd": "2026-07-31",
  "articles": [],
  "summary": "Optional month overview",
  "byCategory": {
    "testing": [],
    "automation": []
  },
  "generatedAt": "2026-07-14T12:00:00Z"
}
```

- `month` — `YYYY-MM` identifier
- `monthStart` / `monthEnd` — inclusive date bounds of the aggregation window (ISO date)
- `articles` — top 20-30 canonical articles ranked by score (DESC), deduplicated by article id
- `summary` — optional human/LLM-generated month overview (may be empty string)
- `byCategory` — required map of category name to the article list for that category (subset of `articles`)
- `generatedAt` — ISO 8601 timestamp of export

Storage: n8n queries `canonical_articles`, groups by week/month, scores/ranks, and exports as JSON. No new database tables are introduced; M5 is a read-and-project layer over M4.

## Work Breakdown

### Track A: Weekly Highlights

- **Task A1:** Create weekly aggregation workflow (n8n SQL-like aggregation)
- **Task A2:** Export weekly JSON to `qa-news/public/weekly-data.json`
- **Task A3:** Build Weekly page in QA News (Next.js `/weekly` route)
- **Task A4:** Test weekly aggregation (sorting, dedup, schema validation)

### Track B: Monthly Recap

- **Task B1:** Create monthly aggregation workflow (n8n SQL-like aggregation)
- **Task B2:** Export monthly JSON to `qa-news/public/monthly-data.json`
- **Task B3:** Build Monthly page in QA News (Next.js `/monthly` route)
- **Task B4:** Test monthly aggregation (sorting, category grouping, schema validation)

Both tracks are independent and can execute in parallel.
